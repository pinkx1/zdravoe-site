# Сайт «Здравое мышление»

Этот репозиторий содержит код WordPress‑сайта «Здравое мышление» (темы, плагины и общую конфигурацию), **без** чувствительных данных: паролей к базе данных, бэкапов и прочего.

## Структура

- `wp-content/themes/zdravoe-theme` — основная тема проекта.
- `wp-content/plugins` — сторонние и/или кастомные плагины.
- `wp-config.example.php` — шаблон конфигурации WordPress **без секретов**.
- `wp-config.php` — реальный конфиг окружения, **игнорируется git’ом**.

## Настройка локальной разработки

1. Создайте сайт в Local (или другом окружении) и укажите папку проекта как корень сайта.
2. Скопируйте `wp-config.example.php` в `wp-config.php` и заполните:
   - `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_HOST` — параметры вашей локальной базы;
   - уникальные ключи и соли (сервис: https://api.wordpress.org/secret-key/1.1/salt/);
   - при необходимости `WP_HOME` и `WP_SITEURL` (локальный домен).
3. Импортируйте дамп боевой базы в локальную БД и обновите домен (search/replace старого домена на локальный).

Файл `wp-config.php`, бэкапы и загруженные файлы (`uploads`) лежат **вне репозитория** и не попадают на GitHub.

## Git и деплой

- Основная ветка: `main`.
- В репозиторий попадает только код (темы, плагины, общие настройки без секретов).
- Деплой на боевой хостинг выполняется либо:
  - вручную (FTP/SFTP, SSH + `git pull`), либо
  - автоматически через GitHub Actions (при пуше в `main`).

Секреты (доступы к БД, FTP, SSH и т.п.) должны храниться либо в непопадающих в репозиторий файлах (`wp-config.php`), либо в секретах GitHub (для CI/CD), но никогда не коммититься в git
